SensoPatch

  • Skills

    - Flex PCB design on KiCAD
    - nRF52 microcontroller
    - Bluetooth LE (6 peripherals)
    - THT/SMD soldering

SensoPatch is a wireless, reconfigurable haptic feedback system designed for study on upper limb amputees with EMG-controlled prosthetic hand. The system consists of a tactile pressure sensing glove and wireless vibration patches.

Design



The Glove

  • 5x5 sensor array on fPCB: 5 conductive column traces and 5 conductive row traces intertwine at major sensory locations in the fingers and palm
  • Zig-zag trace design allowing fPCB to stretch to facilitate flexible movement of the thumb
  • Features XIAO BLE nRF52840, lithium-ion battery and on/off switch
  • Once the Velostat on the fPCB is pressed, the pressure at each intersection directly translates to voltage readings. The pressure on the hand is measured via TDMA approach.

    • The Patches

  • Each features XIAO BLE nRF52840, vibration motor, boost circuit, lithium-ion battery, and on/off switch on a 1.36” diameter PCB.
  • Can be placed anywhere on the body. The modularity of each feedback motor allows users to easily alter the number of motors and customize motor placement.

    • How It Works



    The tactile sensor array collects and wirelessly transmits pressure information to a set of haptic feedback patches that convert the sensor data into vibrotactile sensations for users.

    Since there are more number of sensors than the number of feedback motors, we can program different methods of compressing tactile data into feedback by mapping sensors on different areas of the hand to the vibration motors, as shown in the table. We can also vary the number of the motors and stimulation location to find the optimal configuration that delivers efficient feedback without confusing the users.

    Project Goals

    SensoPatch is a reconfigurable, open-source tool that allows future research to conduct studies on various aspects of the optimal method of haptic feedback delivery. As part of the preliminary study, participants were recruited to complete blindfold discrimination and functional tasks using controlling myoelectric hand via EMG and bypass prosthetic.     The Wizard Game

    • Skills

      - Digital circuit design & assembly
      - Finite State Machine

    As a group project, we design a two player game using two parallel FSMs that interact through combinatorial logic.

    Rules of Game

    Each player has a mana level from 0-5, which is divided into tiers:
    • • Tier I (mana level 1 or 2)
    • • Tier II (mana level 3 or 4)
    • • Tier III (mana level 5)
    Both players start at mana level 0. Each turn, players simultaneously select one of the three actions:
    • • Potion: increment mana by 1, offer no protection nor damage.
    • • Cast: cast a spell on the opponent and reduce the caster’s mana to 0. If both players cast spells at the same time, the play with stronger spell wins. The strength of a spell is dependent on the caster’s mana level. Tier III spells can break through a Block. Tier II spells beat both Tier I and Tier III spells. If both players cast spells in the same tier, the game is tied.
    • • Block: protect a player from a spell (except Tier III spell)
    At the end of every round, the FSM logic counts and displays each player’s mana level and decides if any player has entered a win state. The Wizard Game is played until a winner is decided or the game ties.

    Design

    The circuit consists of flip-flops, combinatorial logic, MUX, decoder, counter, timer chip, 7-segment displays for timer and mana levels, buttons for 3 actions, and reset button.

    Biogaming Controller

    • Skills

      - Arduino
      - Ultrasonic sensor, pressure sensor, IMU

    Individuals with upper limb amputations often face challenges in performing daily task. Gaming included! As a group project, we designed a gaming controller that incorporates sensors to control keyboard and mouse using head and foot tracking.

    Design


    The headband features an IMU and pressure sensor. IMU data is used to control either 2-axis movement or 2-axis camera view (WASD keys) and sprint option (ctrl key). Blowing or sucking into the tube connected to pressure sensor simulates right and left mouse click. To toggle between using the IMU as movement mode or camera view mode, apply the maximum pressure to the tube.

    The foot tracker consists of 2 ultrasonic sensors. Tilting the foot at opposite angles controls jumping (space bar) and crouching (shift key). Rest position indicates no movement.

    With a total of 4 sensors, we achieved 13 distinct controls of the keyboard and mouse. In class competition, our team won the Minecraft obstacle course using this biogaming controller! Below is a demo of Minecraft control.
